## 我们的核心理念

### 1. 虚实共生

我们相信：像对待真实空间一样对待虚拟空间。

### 2. 空间感知体验

让用户在空间中移动时获得完全可信的感知体验。

### 3. 知识网络

知识是相互关联的，形成网络而非线性。

### 4. 智力装备

知识网络是"存储态"，智力装备是"应用态"——困境触发的空投铠甲，按需组装，即时可用。

### 智力装备理论 ⚡ 新增

**核心理念**：从静态知识到动态网络的知识学习范式变革

> "构建一套人工智能时代的'活的知识体系'，打破静态知识的时空限制，让人类的知识实现实时演进、按需部署、普惠共享。"

#### 核心框架

```
静态知识 → 动态知识网络（存储态）→ 智力装备（应用态）
```

运作逻辑：**困境识别 → 知识激活 → 快速组装 → 即时部署 → 反馈迭代**

#### 核心观点

> **"当代知识工作者正身处悖论：知识爆发式增长，获取门槛降至最低，但有效学习的难度并未降低。"**

> **"静态知识'发布即死亡'：一旦发布便无法更新——这与知识动态演进的本质形成根本矛盾。"**

> **"知识网络是存储态，智力装备是应用态。"**

> **"7天入门并非营销噱头，而是基于认知科学设计的跨越转变期的理想周期。"**

> **"转变期是技能习得全周期中风险最高的阶段，绝大多数学习者的技能学习都终止于这一阶段。"**

> **"智力装备的核心定位是'赋能而非替代'。"**

- Kolb体验式学习理论（1984）

- Kieran Egan认知工具理论（1997）

- AI辅助学习研究

#### 五大设计原则

| 原则 | 内容 |
|------|------|
| 最小认知负荷 | 剔除无关信息，渐进复杂度 |
| 成就感反馈 | 微目标拆解、即时验证、进度可视化 |
| 最短路径 | 预定义验证路径，成果优先 |
| 困境预设 | 高频困境急救包，快速检索 |
| 可生长 | 反馈收集、版本控制、协作 |

#### 五大类型

1. **启蒙型** - 无明确目标，建立领域认知
2. **入门型** - 零基础，跨越转变期（7天入门）
3. **工具型** - 掌握特定工具或技术
4. **问题解决型** - 解决具体卡点
5. **项目型** - 完成完整项目

#### 适用边界

**适用**：应用技术类、创意设计类、工程操作类、工具软件类

**不适用**：强监管高风险领域（医学/法律）、需长期积累的基础学科

**核心洞察**：Eevee不是"低质量渲染"，而是"设计工具"

- Eevee的实时性让迭代速度提升10倍
- 光照预览所见即所得
- 材质节点可以实时调整
- 配合几何节点实现程序化生成

#### AI生成心得

**核心洞察**：AI不是替代创作，而是降低门槛

- 图片→3D（Sharp/Meshy）：快速概念验证
- 视频生成（Wan2.2）：动态场景构建
- 本地模型（Ollama）：隐私保护 + 离线可用
- 组合工作流：AI辅助 → 人工调整 → 最终产出

[详情 →](../3d-reconstruction/README.md)

#### 空间计算心得

**核心洞察**：空间计算的核心是"可信度"

- 60fps是沉浸感底线
- 延迟 < 20ms 才能避免晕动症
- 空间锚点让虚拟物体"固定"在真实环境
- 手势交互要符合物理直觉

### 失败案例

#### 过度技术导向

**问题**：追求技术指标，忽视用户体验

**表现**：
- 过度追求画面质量，忽视加载速度
- 引入过多新技术，团队难以驾驭
- 追求"酷炫"效果，忽略实际需求

**教训**：
- 技术服务于体验，不是反过来
- 团队能力决定技术选型
- 需求驱动技术决策

#### Scope Creep

**问题**：项目范围不断膨胀

**表现**：
- 不断添加新功能
- 没有明确的发布标准
- 团队疲劳，质量下降

**教训**：
- 设定明确的MVP
- 学会说"不"
- 迭代优于一步到位

#### 忽视性能优化

**问题**：高估终端设备性能

**表现**：
- 模型面数过高
- 纹理过大
- 特效过于复杂

**教训**：
- 目标设备测试先行
- 性能是功能的一部分
- 烘焙是朋友，不是敌人
