## 二、各版本详细归档文档

### （一）drop-flow-1.0-prototype.md（滴流 1.0 原型版）

#### 1. 基础信息
- **版本定位**：作品技术原型与视觉底层实验版本，核心验证 "实时音画互动" 技术可行性
- **时间范围**：2025.01-04（核心研发）；2025.04.07（发布技术验证视频）；2025.05.14（发布内部展演实录）
- **核心贡献者**：视觉艺术（钱誉文）、电子音乐（徐昊）

#### 2. 核心技术
- **核心框架**：基于 OSC（Open Sound Control）协议 + Blender Python OSC 插件实现实时控制，结合 Blender 几何节点（模拟节点）搭建生成视觉系统
- **技术亮点**：通过 "blender midi + blender geometry node +（audio2bl）sound bake to curves" 实现音频驱动视觉，超过 300 帧的 mesh 点自动删除仍能稳定 60 帧
- **硬件控制**：引入 Akai APC mini 控制器（推子 / 按钮），实现现场实时操控视觉参数，摆脱纯预渲染视频的限制

#### 3. 核心视觉与叙事
- **视觉核心**：纯黑白点线面阵列，将音乐频谱分析转化为立体、持续运动的圆环阵列，以 "点的轨迹流动" 为核心视觉语言
- **音乐创作**：早期音乐暂命名《由一滴水构成的》，核心表达 "由一个点出发的流动声音"，摒弃传统电子乐 "峰值爆发" 逻辑，聚焦 "持续的流动感"
- **作品命名**：确定 "滴流（Drop Flow）"，核心概念为 "一滴水落下后的平静与持续流动"

#### 4. 关键物料
- 技术验证视频（2025.04.07）：https://www.bilibili.com/video/BV12tRrYTEYB
- 内部展演实录（2025.05.14）：2025 VIRTURA | 滴流 Drop Flow | 上海 UFO Terminal 未知终端 | 沉浸式空间实时视觉演出 | 一种持续性的结构涌现

#### 5. 后日谈（非正式记录）
- **创作启发**：音乐声场设计受真锅大度演出启发（其演出后关闭屏幕，用有方向感、高冲击力的声场营造沉浸式体验），该引用仅作为创作背景，不纳入正式表达
- **技术目标**：核心解决 "实时性" 问题，为后续版本的现场演出奠定底层技术基础

### （二）drop-flow-ufo-internal.md（滴流 2.0 UFO内部实验版）

#### 1. 基础信息
- **版本定位**：UFO Terminal「加载… 创作营」专属内部实验展演版本，综合演进版本
- **时间**：2025.05（创作营展演）
- **核心贡献者**：钱誉文（视觉 / 场景，现场）、许思慧（交互艺术，现场）、徐昊（电子音乐，武汉远程）、黄敏萱（AI 视觉，海外远程）；创作支持：UFO Terminal「加载... 计划」

#### 2. 核心技术与视觉拓展
- **技术继承**：沿用 1.0 的实时控制框架（引用技术验证视频：https://www.bilibili.com/video/BV12tRrYTEYB），前 1/3 演出为实时交互的纯生成视觉
- **视觉新增**：
  - 许思慧：基于虚幻 5 粒子系统制作高视觉冲击力的方块类视觉，融入音频分析逻辑
  - 黄敏萱：AI 生成视觉内容，补充 "数字生态 / 人工智能生态" 叙事维度
  - 钱誉文：加入雷克雅未克雏菊苗圃高斯点云扫描素材，制作 "黑白点云→上色→生长" 动画，实现 "点→线→面→高斯 splits→彩色流动" 的视觉演进

#### 3. 核心叙事
- **核心逻辑**："一滴水发展出万物"—— 从纯数字的点线面阵列，到 AI 生成内容，再到现实扫描的自然高斯点云，试图将 "数字 / 现实" 碎片化素材串联为一体
- **表达核心**：仅聚焦 "持续流动、持续涌现"，尚未形成完整的当代艺术语境表达，内容偏多元尝试但通过音乐串联

#### 4. 后日谈（非正式现场记录）
- **现场意外**：后半段预渲染视频素材因 Blender 加载路径错误导致全屏紫屏，调试两次未果后切换备用视频，观众误认是设计效果
- **协作状态**：仅钱誉文、许思慧到场执行，徐昊（武汉）远程输出音乐，黄敏萱（海外）远程提供素材
- **技术痛点**：Blender 长时间播放预渲染视频易卡顿，暴露 "视觉系统标准化" 需求

### （三）drop-flow-3.0-roooooom719.md（滴流 3.0 展演版）

#### 1. 基础信息
- **版本定位**：作品成熟公开演出版本，PRE（夏天老师）邀请创作的 Rooooooom719 音画现场专属版本
- **时间**：2025.07
- **核心贡献者及角色设定**：
  - 钱誉文：视觉艺术 / 场景构成（主厨・能量校准）
  - 马逍遥：视觉艺术（信息研究员・数据采样，昵称 "马蜥蜴"）
  - 徐昊：电子音乐（声呐雷达手・音乐坐标）
  - 许思慧：声光装置（护士・信号急救）

#### 2. 核心技术与视觉
- **技术升级**：优化实时互动系统，实现音频分析与视觉参数全维度映射，每场演出视觉为独一无二的实时生成结果
- **空间光影**：受真锅大度地下空间个展作品启发（后日谈），强化 360° 环形屏 "边界消融" 的包裹式沉浸感
- **视觉新增（马逍遥）**：像素化内容、深海怪物 / 蜥蜴骨架视觉元素（呼应 "马蜥蜴" 昵称）

#### 3. 核心叙事与演出设计
- **沉浸式设定**："数字信息流潜艇乘务组"—— 带领观众 "潜入数字信息流"
- **演出桥段**：
  - 开场：红灯闪烁（报警）→ 从休眠状态唤醒，背景粒子飞动
  - 核心：360 度相机拍摄的近距离眼睛画面（钱誉文连夜制作），营造 "所有人被装在罐子中被外部注视" 的沉浸感
  - 视觉转换：继承 timer 展览（2025.11-12）的高斯数字花园理念，实现 "点→现实空间" 的转换（两个三角形变形），"一滴水的声响" 为空间引入巨型物体，打造雾气弥漫的空间氛围

#### 4. 后日谈（非正式记录）
- **反馈与启发**：现场演出时长仅 6 分钟，被反馈 "时长过短"，埋下 "搭建标准化视觉系统" 的种子
- **创作背景**：空间光影编排受真锅大度作品启发，未直接引用，仅为创作参考

### （四）drop-flow-hangzhou.md（滴流 杭州竞赛 / 双年展版）

#### 1. 滴流 竞赛获奖版（2025.09）
- **核心定位**：杭州国际电子音乐作曲比赛 C 组一等奖参赛版本
- **适配调整**：精简叙事结构，强化徐昊的电子音乐核心表达，视觉适配比赛场地硬件兼容性
- **贡献者**：电子音乐（徐昊，武汉音乐学院）、视觉艺术（钱誉文）

#### 2. 滴流 杭州双年展版（2025.10）
- **版本定位**：首届中国（杭州）艺术与科技国际双年展（国家级国际展览）开幕式演出 + 常设展览版本
- **核心呈现形式**：
  - **开幕式演出（沉浸式肢体交互）**：
    - **技术支撑**：中国美术学院提供动作捕捉设备与排练空间，上海 / 杭州两地排练
    - **核心理念**："从屏幕框定时代到无边际媒体时代"—— 现实空间与数字空间叠加，以 "球形全景视频" 象征 "可包裹、可缩放的无边际媒体内容"
    - **演出流程**：
      - 开场：魏琴（表演艺术家）戴 VR 头盔坐在 LED 屏前，屏幕碎裂→球形全景视频（滴流音画互动内容）撞开屏幕
      - 交互：球体围绕表演者快速运动，表演者与球体互动（查看、将头伸入球体），模拟 "潜入数字空间"
      - 冲突：表演者因数字空间刺激呈现 "焦虑挣扎" 状态（映射当代人对科技 / AI 的焦虑）
      - 和解：空间安静，生长绿色元素（青海自然地理扫描点云素材），团队成员佩戴 AR/VR/Apple Vision Pro 上台，表达 "掌控数字力量、与数字生态共生" 的核心
  - **常设展览**：7 分钟固定影像版本，保留核心叙事与 "数字自然共生" 表达
- **创作团队**：沈轩（策展 / 艺术总监）、王家明（剧场导演）、钱誉文（数字艺术）、魏琴（表演）、徐昊（电子音乐）、许思慧（交互艺术）

### （六）drop-flow-2026.md（滴流 2026 规划版）

## 三、核心总结

- **版本迭代逻辑**：从 "技术验证（1.0）→多媒介实验（2.0）→公开定型（3.0）→赛事 / 展览适配（竞赛 / 双年展 / 重庆版）"，2026 版聚焦技术与交互升级，核心始终围绕 "持续流动""数字与自然共生"
- **技术核心**：以 OSC+Blender 实时控制为底层，从纯数字点线面拓展至现实扫描高斯点云、全景视频，最终实现 "数字 - 现实空间融合"
- **表达演进**：从 "无明确艺术语境的流动感"，到 "数字焦虑→数字共生" 的完整叙事，锚定 "无边际媒体时代的人机关系" 核心议题
