## 核心理念

**在Drop之后，并不必然迎来爆炸，而可能是一种持久的流动，与一场关于感知的平静觉醒。**

这是所有滴流版本万变不离其宗的核心：音乐drop之后的状态，不是激烈的峰值狂欢，而是激烈过后的空缺、下坠后的平静，所有的画面、声音、情绪，都从这一滴坠落的水出发，逐层渗透、持续流动，最终又归于寂静，完成一场从无到有、再归于无的完整闭环。

## 二、各版本详细归档文档

### （一）滴流 1.0 原型版

**版本定位**：作品技术原型与视觉底层实验版本，核心验证 "实时音画互动" 技术可行性

**时间范围**：2025.01-04（核心研发）；2025.04.07（发布技术验证视频）；2025.05.14（发布内部展演实录）

**核心贡献者**：视觉艺术（钱誉文）、电子音乐（徐昊）

**核心技术**：
- 基于 OSC（Open Sound Control）协议 + Blender Python OSC 插件实现实时控制，结合 Blender 几何节点（模拟节点）搭建生成视觉系统
- 通过 "blender midi + blender geometry node +（audio2bl）sound bake to curves" 实现音频驱动视觉，超过 300 帧的 mesh 点自动删除仍能稳定 60 帧
- 引入 Akai APC mini 控制器（推子 / 按钮），实现现场实时操控视觉参数

**核心视觉与叙事**：
- 纯黑白点线面阵列，将音乐频谱分析转化为立体、持续运动的圆环阵列
- 音乐暂命名《由一滴水构成的》，核心表达 "由一个点出发的流动声音"，摒弃传统电子乐 "峰值爆发" 逻辑

**关键物料**：
- 技术验证视频（2025.04.07）：https://www.bilibili.com/video/BV12tRrYTEYB
- 内部展演实录（2025.05.14）

**后日谈**：创作启发受黑川良一关闭屏幕后通过声场和爆闪灯演出启发

[查看完整文档](./drop-flow-1.0-prototype.md)

## 三、核心总结

**版本迭代逻辑**：从 "技术验证（1.0）→多媒介实验（2.0）→公开定型（3.0）→赛事 / 展览适配（竞赛 / 双年展 / 重庆版）"，2026 版聚焦技术与交互升级，核心始终围绕 "持续流动""数字与自然共生"

**技术核心**：以 OSC+Blender 实时控制为底层，从纯数字点线面拓展至现实扫描高斯点云、全景视频，最终实现 "数字 - 现实空间融合"

**表达演进**：从 "无明确艺术语境的流动感"，到 "数字焦虑→数字共生" 的完整叙事，锚定 "无边际媒体时代的人机关系" 核心议题
